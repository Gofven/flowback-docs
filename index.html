    Introduction

    Welcome to the Flowback API! You can see all the available endpoints to the API.

    We currently only have language bindings in Shell. You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    To authorize, use this code:

    curl "api_endpoint_here" \
-H "Authorization: your_token_here"

    Make sure to replace your_token_here with your API key.

    Flowback uses API keys to allow access to the API. You can get an API key using the Login API endpoint.

    Flowback expects for the API key to be included in all API requests to the server (aside from registration) in a header that looks like the following:

    Authorization: your_token_here

    You must replace your_token_here with your personal API key.

    Pagination

    All of the list API endpoints offer pagination support. This feature can be accessed using the query params of the associated API endpoint.

    Every List API will have these parameters in its outermost json object:

    {
  "count": 19,
  "next": "https://v2.flowback.org/api_endpoint?page=2&limit=1&offset=0",
  "previous": false,
  "total_page": 2,
  ...
}

    Query Parameters

    Parameter Default Description
    page int, optional 1 The page of the paginator.
    limit int, optional Varies The maximum amount of entries allowed per page.
    offset int, optional 0 The offset of the paginator, eg. an offset of 2 means the page starts 2 entries ahead per page.

    User

    Register

    This endpoint registers an user to the site. An activation email will be sent to the user containing the verification_code. If FLOWBACK_URL is specified in the backend .env, a link containing the verification_code as a query parameter will be sent instead. The code will be needed for the next step of the registration.

    Example of the link: https://v2.flowback.org/register/

    If FLOWBACK_URL is specified in the backend's .env file, the user will instead receive a link:

    curl "https://v2.flowback.org/register" \
-d "username=flowback" \
-d "email=flowback@example.com"

    HTTP Request

    POST https://v2.flowback.org/register

    Body Parameters

    Parameter Notes
    username string
    email string

    Verify Register

    This endpoint completes an user registration, and creates a password associated to the user.

    curl "https://v2.flowback.org/register/verify" \
-d "verification_code=your_verification_code" \
-d "password=your_password"

    HTTP Request

    POST https://v2.flowback.org/register/verify

    Body Parameters

    Parameter Notes
    verification_code string Can be obtained here.
    password string
    Make sure that the user doesn't do any typing mistakes with the password by e.g. requiring the user to type their password twice.

    Forgot Password

    This endpoint resets the user's password. An activation email will be sent to the user containing the verification_code. If FLOWBACK_URL is specified in the backend .env, a link containing the verification_code as a query parameter will be sent instead. The code will be needed for the next step of the password reset.

    Example of the link: https://v2.flowback.org/forgot_password/

    If FLOWBACK_URL is specified in the backend's .env file, the user will instead receive a link:

    curl "https://v2.flowback.org/forgot_password" \
-d "email=flowback@example.com"

    HTTP Request

    POST https://v2.flowback.org/forgot_password

    Body Parameters

    Parameter Notes
    email string

    Verify Forgot Password

    This endpoint resets the user's password.

    curl "https://v2.flowback.org/forgot_password/verify" \
-d "verification_code=verification_code" \
-d "password=password"

    HTTP Request

    POST https://v2.flowback.org/forgot_password/verify

    Body Parameters

    Parameter Notes
    verification_code string Can be obtained here.
    password string
    Make sure that the user doesn't do any typing mistakes with the password by e.g. requiring the user to type their password twice.

    List Users

    This endpoint gets a list of users.

    curl "https://v2.flowback.org/users" \
-H "Authorization: your_token_here"

    The above command returns JSON structured like this:

    {
  "data": [
    {
      "id": 1,
      "username": "flowback",
      "profile_image": "https://example.com/media/image.png"
    },
    ...
  ]
}

    HTTP Request

    GET https://v2.flowback.org/users

    URL Parameters

    Parameter Notes
    username string, optional

    Get User

    This endpoint gets an user.

    curl "https://v2.flowback.org/user/<id>" \
-H "Authorization: your_token_here"

    The above command returns JSON structured like this:

    {
  "data": {
    "id": 1,
    "email": "flowback@example.com",
    "username": "flowback",
    "profile_image": "https://example.com/media/image.png",
    "banner_image": "https://example.com/media/banner.png",
    "bio": "Hello world!",
    "website": "https://example.com/"
  }
}

    HTTP Request

    GET https://v2.flowback.org/user

    URL Parameters

    Parameter Notes
    id int, optional Leave it blank to get the current user.

    Update User

    This endpoint updates an user.

    curl "https://v2.flowback.org/user/update" \
-H "Authorization: your_token_here" \
-d "username=flowback" \
-F "profile_image=@profile_image.png" \
-F "banner_image=@banner_image.png" \
-d "bio=Welcome to my profile!"
-d "website=https://example.com/"

    HTTP Request

    POST https://v2.flowback.org/user

    Body Parameters

    Parameter Notes
    username string, optional
    profile_image file, optional
    banner_image file, optional
    bio int, optional
    website int, optional

    Group

    List Groups

    curl "https://v2.flowback.org/group/list" \
-H "Authorization: your_token_here"

    The above command returns JSON structured like this:

    {
  "data": [
    {
      "id": 1,
      "created_by": 1,
      "active": true,
      "direct_join": false,
      "name": "Flowback",
      "description": "Flowing seamlessly",
      "image": "https://example.com/media/image.png",
      "cover_image": "https://example.com/media/banner.png",
      "joined": true
    },
    ...
  ]
}

    Lists all available groups to join or request invite for

    HTTP Request

    GET https://v2.flowback.org/group/list

    Query Parameters

    Parameter Notes
    id int, optional
    name str, optional Must be exact
    name__icontains str, optional name contains, case-insensitive
    direct_join bool, optional Determines if users can join directly or require invite
    joined bool, optional User joined the group or not

    Detail Group

    curl "https://v2.flowback.org/group/1/detail" \
-H "Authorization: your_token_here"

    The above command returns JSON structured like this:

    {
  "data": {
    "id": 1,
    "created_by": 1,
    "active": true,
    "direct_join": false,
    "public": true,
    "default_permission": 3,
    "name": "Flowback",
    "description": "Flowing seamlessly",
    "image": "https://example.com/media/image.png",
    "cover_image": "https://example.com/media/banner.png",
    "jitsi_room": "https://meet.example.com/ABC123"
  }
}

    Lists all available groups to join or request invite for

    HTTP Request

    GET https://v2.flowback.org/group/<id>/detail

    Create group

    curl "https://v2.flowback.org/group/create" \
-H "Authorization: your_token_here" \
-d "name=Flowback" \
-F "description=Flowing seamlessly" \
-F "image=@image.png" \
-F "cover_image=@banner_image.png" \
-d "direct_join=true" \
-d "public=true"

    Creates a group

    HTTP Request

    POST https://v2.flowback.org/group/create

    Query Parameters

    Parameter Notes
    name int Name of the group
    description str Description of the group
    image img Image of the group
    cover_image img Banner image of the group
    direct_join bool Determines if users can join directly or require invite
    public bool Determines the visibility of the group

    Update group

    curl "https://v2.flowback.org/group/update" \
-H "Authorization: your_token_here" \
-d "name=Flowback" \
-F "description=Flowing seamlessly" \
-F "image=@image.png" \
-F "cover_image=@banner_image.png" \
-d "direct_join=true" \
-d "public=true" \
-d "default_permission=1"

    Updates a group

    HTTP Request

    POST https://v2.flowback.org/group/update

    Query Parameters

    Parameter Notes
    name int, optional Name of the group
    description str, optional Description of the group
    image img, optional Image of the group
    cover_image img, optional Banner image of the group
    direct_join bool, optional Determines if users can join directly or require invite
    public bool, optional Determines the visibility of the group
    default_permission int, optional Sets the default permissions for users, allows null value for basic permissions

    Delete group

    curl "https://v2.flowback.org/group/delete" \
-H "Authorization: your_token_here"

    Deletes a group

    HTTP Request

    POST https://v2.flowback.org/group/delete

    List users

    curl "https://v2.flowback.org/group/1/users" \
-H "Authorization: your_token_here"

    The above command returns JSON structured like this:

    {
  "data": [
    {
      "user_id": 1,
      "username": "User",
      "profile_image": "https://example.com/media/image.png",
      "banner_image": "https://example.com/media/banner.png",
      "delegate": 2,
      "permission_id": 1,
      "permission_name": "Bakery"
    }
  ],
  ...
}

    Lists all users in the group

    HTTP Request

    GET https://v2.flowback.org/group/<id>/users

    Query Parameters

    Parameter Notes
    group_user_id int, optional Unique identifier for the group user
    user_id int, optional
    username__icontains str, optional Username contains (incase-sensitive)
    delegate bool, optional Whether user is a delegate or not
    is_admin bool, optional
    permission int, optional The permission ID of the user

    Join group

    curl "https://v2.flowback.org/group/<id>/join" \
-H "Authorization: your_token_here"

    The above command returns JSON structured like this:

    {
  "data": "join"
}

    Join a group, this will return either "join", or "invite" depending on the group settings

    HTTP Request

    POST https://v2.flowback.org/group/<id>/join

    Leave group

    curl "https://v2.flowback.org/group/<id>/leave" \
-H "Authorization: your_token_here"

    Leaves a group

    HTTP Request

    POST https://v2.flowback.org/group/<id>/leave
